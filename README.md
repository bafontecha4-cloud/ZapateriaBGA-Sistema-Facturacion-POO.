##  Descripción del Proyecto
Desarrollar una aplicación de escritorio en Java SE con Swing y SQLite que permita gestionar de manera eficiente las ventas, clientes, productos y usuarios de un negocio de calzado, ofreciendo una interfaz clara y segura para el manejo diario.

**Objetivos Específicos**
Implementar un sistema de inicio de sesión con roles y permisos diferenciados.
Construir módulos CRUD para clientes, productos, usuarios y ventas.
Diseñar una interfaz gráfica intuitiva y moderna que facilite la navegación.
Integrar reportes de ventas con filtros por cliente y factura.
Garantizar la persistencia de datos mediante conexión JDBC a SQLite.
Organizar el proyecto aplicando principios de Programación Orientada a Objetos (POO).

**Justificación**
El proyecto nace como una solución práctica para mejorar el manejo de ventas en un negocio de calzado. Se busca optimizar la gestión interna mediante un sistema que permita registrar clientes, productos, ventas y usuarios de forma organizada, rápida y segura. La aplicación está pensada para negocios que no dependen de conexión a internet constante, ofreciendo una herramienta sencilla pero completa para apoyar la gestión diaria.

**Entorno de Trabajo**
Lenguaje: Java SE
GUI: Swing (JFrame, JPanel, JMenu, JTable, etc.)
Base de Datos: SQLite
IDE: NetBeans 28
Control de Versiones: GitHub

**Requerimientos Funcionales**
Inicio de sesión con validación de credenciales y roles.
Menú principal con navegación entre módulos.
CRUD completo para clientes, productos, usuarios y ventas.
Generación de reportes filtrados por cliente y factura.
Opción de cierre de sesión seguro.

**Requerimientos No Funcionales**
Interfaz gráfica clara y fácil de usar.
Seguridad en el manejo de credenciales de usuario.
Persistencia confiable de datos en SQLite.
Modularidad para facilitar futuras mejoras.
Portabilidad: ejecutable en cualquier sistema con Java instalado.
